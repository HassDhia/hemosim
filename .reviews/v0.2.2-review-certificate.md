## Resolved Findings (context — landed in Phase C, not re-litigated in Phase D)

Referenced for completeness; the Phase-D review does not re-open them. Phase-D assessment confirms each resolution is defensible as landed.

1. **`heparin_raschke_aptt` — fitted-params test path.** Phase-C resolution: the test now loads `results/published_calibration.json::heparin_fit.fitted_params` rather than using unfitted HeparinPKPD defaults. The paper's 75 s cohort-mean claim is tied to the fitted params and reproduces to machine precision (residual ≈ 0.000 s). Paper unchanged. **Phase-D assessment: defensible** — a pharmacist peer would recognize fitted-params reproduction as the correct measurement path for a calibration-harness claim.

2. **`dic_hockin_mann_cascade` — thrombin activation registry rescope.** Phase-C resolution: the original Phase-A target ("100–400 nM at 5–15 min per Hockin-Mann 2002 Fig 3") was pulled from the full 34-species source model and exceeded what paper §6 actually commits to. Registry corrected to a qualitative scope (thrombin > 1 nM within 30 min under DIC-consistent AT-III depletion + strong trigger); test uses that scope and passes. Paper unchanged. **Phase-D assessment: defensible** — the weakest-credible-commitment authoring discipline (captured in `feedback/2026-04-18-sr-m-registry-authoring-discipline.md`) is the correct principle; the paper never claimed amplitude reproduction, and the registry should not have either.

3. **`dic_hockin_mann_cascade` — mass-conservation invariant correction.** Phase-C resolution: the original test used wrong numpy axis indexing and the wrong invariant (sum-of-all-8-states, but the 8-state reduction is open by design — prothrombin is a parameter, AT-III is an infinite pool). Test rewritten to measure fibrinogen+fibrin conservation only (the one invariant the reduction preserves by construction) with correct axis. Drift < 0.005 over 30 min. Paper unchanged. **Phase-D assessment: defensible** — the paper §6 does not claim mass conservation of the reduced system; inventing a conservation invariant that the reduction is not designed to preserve was a Phase-A authoring bug, now corrected.

4. **Nemati → Wan string-field propagation.** Phase-C opportunistic fix: zero residual `nemati_ttr_standard` tokens across `src/hemosim/validation/published_calibration.py`, `results/published_calibration.json`, and `tests/test_published_calibration.py`. Paper §8 already correctly labeled this as Wan 2008 from v0.2.1 H4. **Phase-D assessment: defensible** — the code now matches the paper's corrected narrative end-to-end.
